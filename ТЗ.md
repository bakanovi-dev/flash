# Техническое задание: Приложение для изучения английских фраз

## 1. Обзор продукта

Мобильное приложение (iOS + Android) для изучения английских фраз, идиом и выражений в контексте. Главный экран — лента рилов (текстовых карточек), персонализированная под интересы пользователя. Контент автоматически размечается LLM по 5 осям и хранится в MongoDB.

**Целевая аудитория:** люди уровня B1–C2, которые хотят учить живой английский из реальных источников (сериалы, книги, подкасты).

**Языки интерфейса и объяснений (на старте):** русский (ru), французский (fr), немецкий (de), итальянский (it), китайский (zh). Архитектура рассчитана на добавление новых языков без изменения схемы данных.

#### Экран: Лента рилов (главная)

**Header:**
- Логотип (по центру или слева)
- Иконка бургер-меню (справа)

**Карточка рила (полный экран, свайп вверх/вниз):**
- Фраза (крупно, основной текст)
- Контекстная цитата (курсив, меньший шрифт)
- Источник (название сериала/книги, эпизод)
- Базовое объяснение (бесплатно)
- Блок «Подробнее» (заблюрен для free-пользователей, клик → paywall)
- Слова из цитаты с уровнем CEFR (например: «aware — B1»)
- Кнопка «Сохранить» (иконка закладки)

**Жесты и действия:**
- Свайп вверх → следующая карточка (записывается как `skip`, если карточка не была открыта)
- Свайп вниз → предыдущая карточка
- Тап по закладке → сохранить / убрать из сохранённых (`save`)
- Тап по кнопке лайк → `like`
- Тап по кнопке дизлайк → `dislike`

**Логика подбора карточек:**
- MVP: фильтрация по `preferences.domains` пользователя + случайная выборка
- Исключаются уже показанные и сохранённые карточки текущей сессии
- Пагинация: подгружать батчами по 10

| Функция | Free | Premium |
|---|---|---|
| Лента рилов | ✅ | ✅ |
| Перевод цитаты | ✅ | ✅ |
| Сохранение карточек | ✅ | ✅ |

| Остальные выражения (literal + explanation) | ❌ (paywall) | ✅ |
| Словарные слова с переводом | ❌ (paywall) | ✅ |

**Что видит free-пользователь на карточке:**
- Цитата на английском
- Перевод цитаты на свой язык
- Первое выражение (≈5% контента) — полностью открыто, как тизер
- Остальные выражения и все слова — заблюрены → тап → paywall

## 7. Backend API

### Аутентификация
Все запросы (кроме публичных) требуют `Authorization: Bearer <firebase_token>` в заголовке. Backend верифицирует токен через Firebase Admin SDK.

### Эндпоинты

#### Пользователи
```
POST   /api/v1/users                    — создать профиль после регистрации
GET    /api/v1/users/me                 — получить свой профиль
PATCH  /api/v1/users/me                 — обновить профиль (интересы, имя)
DELETE /api/v1/users/me                 — удалить аккаунт
```

#### Лента
```
GET    /api/v1/feed                     — получить батч карточек
       ?limit=20&exclude_ids=[...]      — параметры
```

#### Рилы
```
GET    /api/v1/reels/:id                — получить один рил
GET    /api/v1/reels/:id/advanced       — расширенное объяснение (Premium)
```

#### Сохранённые карточки
```
GET    /api/v1/saved                    — список сохранённых
POST   /api/v1/saved/:reel_id           — сохранить
DELETE /api/v1/saved/:reel_id           — убрать из сохранённых
```

#### Взаимодействия (для персонализации)
```
POST   /api/v1/interactions             — залогировать действие
       body: { reel_id, action }        — action: like | dislike | save | skip
```

#### Контент
```
POST   /api/v1/contact                  — форма обратной связи
POST   /api/v1/partners                 — заявка от партнёра
```

## 8. LLM Content Pipeline

### 8.1 Поддерживаемые форматы входных файлов

На старте поддерживается только `.srt`. Остальные форматы — следующие итерации.

| Формат | Статус |
|---|---|
| `.srt` | ✅ MVP |
| `.txt` | Фаза 2 |
| `.pdf` | Фаза 2 |
| `.epub` | Фаза 2 |
| `.fb2` | Фаза 2 |

### 8.2 Этапы пайплайна

**Шаг 1: Парсинг файла**
- SRT: извлекаем текст с временными метками, чистим от тегов форматирования
- PDF/EPUB/FB2: конвертируем в plain text (библиотеки: `pdfplumber`, `ebooklib`)
- Разбиваем на предложения/абзацы с сохранением контекста (окно ±2 предложения)

**Шаг 2: Извлечение фраз**
- LLM-промпт: из абзаца извлечь все нетривиальные фразы (идиомы, коллокации, термины, фразовые глаголы)
- Формат ответа: массив `{phrase, quote, context_before, context_after}`
- Дедупликация по fingerprint (нормализованный lowercase текст фразы)

**Шаг 3: Авторазметка**

Языки генерации задаются параметром `--languages` (по умолчанию: `ru`). Можно передать один язык или несколько через запятую. Каждый язык — отдельный LLM-вызов (параллельно).

Промпт (жёсткий, controlled vocabulary):
```
Source: "{show}", season {season}, episode {episode}
Quote (EN): "{quote_en}"
Expressions found: {phrases_list}

For each expression provide literal translation and explanation.
For the whole reel provide: context description, quote translation, advanced explanation.
Use languages: ru, fr, de, it, zh.

Return JSON:

{
  "locales": {
    "ru": {
      "context": "...",                   // описание сцены, 1-3 предложения
      "quote": "..."                      // перевод цитаты
    },
    "fr": { "context": "...", "quote": "..." },
    "de": { "context": "...", "quote": "..." },
    "it": { "context": "...", "quote": "..." },
    "zh": { "context": "...", "quote": "..." }
  },
  "expressions": [
    {
      "phrase": "Big bucks",
      "locales": {
        "ru": { "literal": "...", "explanation": "..." },
        "fr": { "literal": "...", "explanation": "..." },
        "de": { "literal": "...", "explanation": "..." },
        "it": { "literal": "...", "explanation": "..." },
        "zh": { "literal": "...", "explanation": "..." }
      }
    }
  ],
  "words": [
    {
      "word": "aware",
      "level": "B1",                      // один из: A2 | B1 | B2 | C1 | C2
      "locales": {
        "ru": { "translation": "..." },
        "fr": { "translation": "..." },
        "de": { "translation": "..." },
        "it": { "translation": "..." },
        "zh": { "translation": "..." }
      }
    }
  ],
  "tags": {
    "domains": [],                        // 1-3 из controlled vocabulary
    "emotion": "",                        // joy | anger | sarcasm | confidence | fear | nostalgia | neutral
    "register": "",                       // vulgar | slang | colloquial | neutral | formal | archaic
    "type": "",                           // idiom | phrasal_verb | collocation | term | aphorism | proverb | slang_word
    "cefr": "",                           // A2 | B1 | B2 | C1 | C2  (уровень всего рила)
    "region": "",                         // american | british | australian | neutral
    "era": ""                             // contemporary | 20th_century | archaic
  },
  "social_risk": false,
  "adult": false
}
```

**Шаг 4: Генерация эмбеддинга**
- Управляется флагом `--embedding` (по умолчанию: выключено)
- Модель: `text-embedding-3-large` (OpenAI) или аналог — задаётся в конфиге
- Вход: `"{quote_en} {expressions_phrases_joined}"`
- Результат сохраняется в поле `embedding`

**Шаг 5: Загрузка в MongoDB**
- Статус `"pending"` до прохождения QA
- После одобрения в QA UI → статус `"published"`

### 8.3 Запуск

Пайплайн принимает папку (`--dir`) и обрабатывает все `.srt`-файлы внутри по очереди.

```bash
# обработать все .srt из папки, только русский (по умолчанию)
python pipeline.py --dir ./subtitles/silicon_valley_s02 --source "Silicon Valley" --type series --season 2

# с несколькими языками и эмбеддингами
python pipeline.py --dir ./subtitles/silicon_valley_s02 --source "Silicon Valley" --type series --season 2 --languages ru,fr,de --embedding
```

Имя файла используется для определения номера эпизода, если оно соответствует шаблону `s01e03` или `1x03`. Иначе номер эпизода берётся из порядкового номера файла в папке.

## 9. Административный QA-интерфейс

Простое веб-приложение (можно на Streamlit или минимальный React) для проверки результатов пайплайна.

### Функционал:
- Список рилов со статусом `"pending"`
- Карточка рила: цитата (EN), выражения, слова, теги
- Табы по языкам (RU / FR / DE / IT / ZH): context, quote, explanation_advanced + literal/explanation каждого выражения
- Возможность вручную отредактировать любое поле на любом языке
- Кнопки: **Опубликовать** / **Отклонить** / **Редактировать теги**
- Фильтр по источнику, домену, статусу
- Счётчик: обработано / опубликовано / отклонено
- Bulk-публикация (выбрать несколько → опубликовать всё)

## 10. Локализация (i18n)

### Поддерживаемые языки

| Код | Язык | Статус |
|---|---|---|
| `ru` | Русский | Запуск |
| `fr` | Français | Запуск |
| `de` | Deutsch | Запуск |
| `it` | Italiano | Запуск |
| `zh` | 中文 | Запуск |
| ... | Другие | Следующие итерации |

### Что локализуется

1. **UI приложения** — все строки интерфейса через `react-i18next`. Все языки хранятся в **одном файле** `src/locales/translations.json`.
2. **Контент карточек** — `context`, `quote` рила + `literal`/`explanation` каждого выражения + переводы слов — всё хранится в `locales` внутри документа рила. Backend отдаёт только нужный язык по полю `language` из профиля пользователя.

### Структура файла UI-переводов

Все языки в одном файле — добавить или удалить язык = добавить или удалить один ключ:

```json
// src/locales/translations.json
{
```

`react-i18next` инициализируется с этим файлом как единственным ресурсом. При смене языка вызывается `i18n.changeLanguage(lang)`.

### Логика определения языка

1. При первом запуске — `expo-localization.getLocales()` предлагает язык устройства (если он в списке поддерживаемых).
2. Пользователь подтверждает или меняет на шаге 1 онбординга.
3. После смены языка в Профиле — PATCH `/api/v1/users/me` с новым `language`, i18n переключается без перезапуска приложения.

### Добавление нового языка

1. Добавить код языка в `SUPPORTED_LANGUAGES` на бэкенде и в константы мобайла.
2. Добавить новый ключ в `src/locales/translations.json` с переведёнными UI-строками.
3. Запустить пайплайн с расширенным промптом, включающим новый язык — он заполнит `locales.{lang}` для новых рилов.
4. Отдельный скрипт бэкфила для существующих рилов без нового языка.

## 13. Открытые вопросы

1. **In-app purchases**: RevenueCat (бесплатно до $2500 MRR, потом пересмотреть). SDK `react-native-purchases`, бэкенд обновляет `subscription.plan` по вебхуку.
2. **Email-провайдер**: Resend (бесплатно до 3000 писем/мес). Python SDK, отправка из FastAPI по событиям формы обратной связи и заявки партнёра.
3. **LLM-провайдер** для пайплайна: требует сравнительного тестирования на реальных субтитрах. Кандидаты — OpenAI GPT-4o и Claude API. Критерии: качество тегов по controlled vocabulary, точность переводов/объяснений, процент невалидного JSON на выходе.
4. **Деплой бэкенда**: собственный VPS (Грузия). FastAPI + Docker. При масштабировании — пересмотреть в пользу managed-платформы.
5. **Начальный контент**: добавляется вручную администратором через LLM pipeline.
